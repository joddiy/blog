PHP 设计模式 - 命令模式

Jun 06, 2016 in Design pattern

前言

这是一个系列里面的一篇，主要介绍的一些 PHP 下的设计模式，大部分来自于 DesignPatternsPHP 项目，可能会加上《Design Patterns》里面的一些内容。因为都是英文版的，所以翻译的时候会出现一些偏差，不要期待太多噢！

目的

封装调用和解耦。把命令封装成对象，因此你可以用不同的（队列或记录）请求来参数化客户端，同时也支持销毁操作。

包括一个调用者和接收者。Command 类暴露给调用者通用的 Execute 方法，在 Execute 方法中 Command 类调用了接收者的方法。因此，调用者仅仅只需知道 Execute 方法即可，实现了解耦。

另外一个特性是 Undo 方法，可以用来销毁 Execute 方法。

使用场景

  • 通过表现的动作来参数化对象，以面向对象的方式替代 callback 方法
  • Command 对象相比于原始的调用有独立的生命周期，因此可以在不同的时间指定、排队、执行请求
  • 支持 Undo 销毁方法，Command 对象的 Execute 方法可以存储客户端的状态，因此可以定义 Undo 方法来之前执行的 Execute 失效
  • 支持日志记录
  • 在原始操作上封装更高级的操作

结构

参与者

  • Command
    • 声明了 Execute 方法
  • ConcreteCommand
    • 定义了接收者对象和操作之间的绑定
    • 通过调用接收者对应的方法实现了 Execute 方法
  • Client
    • 创建 ConcreteCommand 对象，并且设置接收者
  • Invoker
    • 执行 Command 对象的 Execute 方法
  • Receiver
    • 负责具体操作请求的逻辑
