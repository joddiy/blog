PHP 设计模式 - 备忘录模式

Jun 08, 2016 in Design pattern

前言

这是一个系列里面的一篇，主要介绍的一些 PHP 下的设计模式，大部分来自于 DesignPatternsPHP 项目，可能会加上《Design Patterns》里面的一些内容。因为都是英文版的，所以翻译的时候会出现一些偏差，不要期待太多噢！

目的

不违背封装性的前提下，获取和拓展一个对象内部的状态，使这个对象可以在之后重新存储到这个状态。

使用场景

  • 需要存储对象状态的快照，之后需要更新状态
  • 一个直接的接口来获得状态时会暴露实现的细节和破坏对象的封装性

结构

参与者

  • Memento
    • 存储 Originator 对象的内部状态，Memento 可以根据 Originator 的需求来存储合适数量的内部状态。
    • 只允许 Originator 访问对象，通常有两个接口，Caretaker 看到的是 限制 接口，仅仅只能把 Memento 传递给其他对象；Originator 看到的是 宽松 接口，可以获取更新状态所需的所有数据。理想情况下，只有生产 Memento 的 Originator 对象有权访问 Memento 的内部状态。
  • Originator
    • 创建一个包含当前状态快照的 Memento。
    • 使用 Memento 重新存储内部状态。
  • Caretaker
    • 负责安全持有 Memento
    • 不需要操作或确认 Memento 的数据
