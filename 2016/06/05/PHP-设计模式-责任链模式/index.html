PHP 设计模式 - 责任链模式

Jun 05, 2016 in Design pattern

前言

这是一个系列里面的第一篇，主要介绍的一些 PHP 下的设计模式，大部分来自于 DesignPatternsPHP 项目，可能会加上《Design Patterns》里面的一些内容。因为都是英文版的，所以翻译的时候会出现一些偏差，不要期待太多噢！

目的

解耦请求的发送者和接收者。使用多个类来处理请求，这些类组成请求链。请求在链上传递直到有类可以处理它。

使用场景

  • 超过一个对象来处理请求，而且不知道哪一个会处理
  • 想要某些对象来处理请求而不用特别指定哪一个
  • 处理请求的对象应该被动态指定

结构

一个典型的结构如下：

参与者

  • Handler
    • 定义处理请求的接口
    • （可选）实现继承的引用
  • ConcreteHandler
    • 如果可响应的话就处理请求
    • 访问继承者
    • 如果能处理请求就处理，不能就传给继承者
  • Client
    • 初始化请求链上的 ConcreteHandler 对象

总结

责任链模式的优点如下：

  • 降低耦合。
  • 指派对象响应方面增加灵活性。
  • 接收不被保证。
