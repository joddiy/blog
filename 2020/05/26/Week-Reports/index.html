what have been done

Any issue encounter

Just like I declared at this issue

  • ONNX prefers to use tensors as input instead of attributes, which may incurs some issues when we create SINGA operators(or layers). There are two cases: SINGA params <- ONNX Initializer and NGA params <- ONNX operator(dynamic graph). We can support first one, but the seconde is hard to support.
  • For some layers in SINGA, there are some bugs needed to fix.
  • We discussed last time to use Metaclass in SONNX, but I check it carefully, we don’t need to use it.

And all these issues has been fixed.
